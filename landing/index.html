Multi-agent workflow orchestration for OpenClaw

Define pipelines in YAML. Install with one command. Let coordinated AI agents plan, implement, verify, test, and ship — autonomously.

$ Install GitHub.com/snarktank/antfarm

Tell your OpenClaw agent to install Antfarm. That's it.

How it works

1

Define

Write a workflow.yml with agents and steps. Each agent gets its own persona, instructions, and workspace.

2

Install

One command provisions agent workspaces, registers them in OpenClaw, and starts cron polling.

3

Run

Give it a task. Agents claim steps, do the work in isolated sessions, pass context forward, and hand off to the next agent.

Built on the Ralph loop

Each agent runs in a fresh session with clean context. Memory persists through git history and progress files — the same autonomous loop pattern from Ralph, scaled to multi-agent pipelines.

Bundled workflows

feature-dev

7 agents

Full development pipeline. A planner decomposes tasks into user stories, then agents implement each one in isolated sessions — verify after every story, integration tests at the end, automatic PR and code review.

plan→ setup→ implement→ verify→ test→ PR→ review

bug-fix

6 agents

End-to-end bug resolution. Triager reproduces the issue, investigator traces root cause, fixer writes the patch with a regression test, verifier confirms it, PR agent ships it. Verify failures loop back automatically.

triage→ investigate→ setup→ fix→ verify→ PR

security-audit

7 agents

Automated security scanning and patching. Scanner checks for vulnerabilities, prioritizer ranks findings, fixer patches each one with tests, verifier confirms, then integration tests and a PR with the full audit report.

scan→ prioritize→ setup→ fix→ verify→ test→ PR

Security

Workflows run agents on your machine. We take that seriously.

Curated repo only

Antfarm only installs workflows from the official snarktank/antfarm repository. No arbitrary remote sources.

Reviewed for prompt injection

Every workflow is reviewed for prompt injection attacks and malicious SKILL.md or AGENTS.md files before merging.

Community contributions welcome

Want to add a workflow? Submit a PR. We encourage it. All submissions go through careful security review before they ship.

Transparent by default

Every workflow is plain YAML and Markdown. You can read exactly what each agent will do before you install it.

Dashboard

Monitor runs, track step progress, and view agent output in real time.

Commands

Lifecycle

antfarm installInstall all bundled workflows
antfarm uninstallFull teardown (agents, crons, DB)

Workflows

antfarm workflow run <id> <task>Start a run
antfarm workflow status <query>Check run status
antfarm workflow runsList all runs
antfarm workflow resume <run-id>Resume a failed run

Management

antfarm workflow listList available workflows
antfarm workflow install <id>Install a single workflow
antfarm workflow uninstall <id>Remove a single workflow
antfarm dashboardStart the web dashboard
antfarm logsView recent log entries

Architecture

Agents poll for work via OpenClaw cron jobs (every 15 min, staggered).
No central orchestrator. Each agent is autonomous.

                ┌─────────┐
                │  SQLite  │  Runs, steps, stories, context
                └────┬────┘
                     │
        ┌────────────┼────────────┐
        │            │            │
   ┌────┴────┐ ┌────┴────┐ ┌────┴────┐
   │ Agent A │ │ Agent B │ │ Agent C │  Each in isolated OpenClaw session
   │ (cron)  │ │ (cron)  │ │ (cron)  │  Fresh context per step/story
   └─────────┘ └─────────┘ └─────────┘

Context flows forward via KEY: value pairs in agent output,
stored as JSON in the run record and templated into the next step.

Create your own

Define agents and steps in YAML. Antfarm handles provisioning, scheduling, and orchestration.

workflow.yml 
id: my-workflow
name: My Custom Pipeline
agents:
  - id: researcher
    name: Researcher
    workspace:
      files:
        AGENTS.md: agents/researcher/AGENTS.md

steps:
  - id: research
    agent: researcher
    input: |
      Research {{task}} and report findings.
      Reply with STATUS: done and FINDINGS: ...
    expects: "STATUS: done"
